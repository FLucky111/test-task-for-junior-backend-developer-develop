# Task Service (Test Task)

## Описание

Реализована поддержка периодических задач в существующем API трекера задач; <br>
Добавлена возможность создавать шаблоны задач с правилами повторения, на основе которых автоматически генерируются обычные задачи.

## Запуск проекта

### 1. Установить PostgreSQL

Создать базу: <br>
CREATE DATABASE taskservice;

### 2. Применить миграции

psql -U postgres -d taskservice -f migrations/0001_create_tasks.up.sql <br>
psql -U postgres -d taskservice -f migrations/0002_create_task_templates.up.sql

### 3. Установить переменную окружения

$env:DATABASE_DSN="postgres://postgres:password@localhost:5432/taskservice?sslmode=disable"

### 4. Запустить сервер

go run ./cmd/api

### 5. Swagger

http://localhost:8080/swagger/

## Основные изменения

### 1. Добавлены сущности

task_templates - шаблоны задач; <br>
task_recurrences - правила повторения.

### 2. Поддержка периодичности

Реализованы типы: <br>
daily - каждые N дней; <br>
monthly - по числу месяца; <br>
specific_dates - конкретные даты; <br>
day_parity - чётные / нечётные дни. 

### 3. Генерация задач

При создании шаблона: <br>
Автоматически создаются задачи в таблице tasks; <br>
Горизонт генерации: 30 дней вперёд.

### 4. Расширение модели задачи

Добавлены поля: <br>
scheduled_for - дата выполнения; <br>
template_id - ссылка на шаблон. 

## API

### Создание шаблона

POST /api/v1/task-templates <br>

Пример: <br>

{ <br>
  "title": "Обзвон пациентов", <br>
  "status": "new", <br>
  "active": true, <br>
  "recurrence": { <br>
    "type": "daily", <br>
    "start_date": "2026-04-06", <br>
    "every_n_days": 2 <br>
  } <br>
}

### Обычные задачи

POST /api/v1/tasks <br>
<br> 
{ <br>
  "title": "Разовая задача", <br>
  "scheduled_for": "2026-04-10" <br>
}

## Принятые решения

### 1. Генерация задач вперёд

Выбран горизонт 30 дней: <br>
Упрощает реализацию; <br>
Не перегружает БД. 

### 2. Хранение recurrence отдельно

Позволяет: <br>
Гибко расширять логику; <br>
Не усложнять таблицу tasks. 

### 3. JSONB для specific_dates

Позволяет хранить массив дат без отдельной таблицы.

### 4. template_id nullable

Позволяет: <br>
Отличать обычные задачи от периодических; <br>
Не ломать существующий функционал.

## Ограничения

Нет фонового обновления задач (cron); <br>
Генерация только при создании шаблона; <br>
Нет обновления уже сгенерированных задач при изменении шаблона.

## Что можно улучшить

Добавить background worker (cron); <br>
Поддержка редактирования шаблонов; <br>
Пересоздание задач при изменении правил; <br>
Пагинация задач.
