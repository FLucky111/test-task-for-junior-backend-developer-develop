# Task Service (Test Task)

## Описание

Реализована поддержка периодических задач в существующем API трекера задач;
Добавлена возможность создавать шаблоны задач с правилами повторения, на основе которых автоматически генерируются обычные задачи.

## Запуск проекта

### 1. Установить PostgreSQL

Создать базу:
CREATE DATABASE taskservice;

### 2. Применить миграции

psql -U postgres -d taskservice -f migrations/0001_create_tasks.up.sql
psql -U postgres -d taskservice -f migrations/0002_create_task_templates.up.sql

### 3. Установить переменную окружения

$env:DATABASE_DSN="postgres://postgres:password@localhost:5432/taskservice?sslmode=disable"

### 4. Запустить сервер

go run ./cmd/api

### 5. Swagger

http://localhost:8080/swagger/

## Основные изменения

### 1. Добавлены сущности

task_templates - шаблоны задач;
task_recurrences - правила повторения.

### 2. Поддержка периодичности

Реализованы типы:
daily - каждые N дней;
monthly - по числу месяца;
specific_dates - конкретные даты;
day_parity - чётные / нечётные дни.

### 3. Генерация задач

При создании шаблона:
Автоматически создаются задачи в таблице tasks;
Горизонт генерации: 30 дней вперёд.

### 4. Расширение модели задачи

Добавлены поля:
scheduled_for - дата выполнения;
template_id - ссылка на шаблон.

## API

### Создание шаблона

POST /api/v1/task-templates

Пример:

{
  "title": "Обзвон пациентов",
  "status": "new",
  "active": true,
  "recurrence": {
    "type": "daily",
    "start_date": "2026-04-06",
    "every_n_days": 2
  }
}

### Обычные задачи

POST /api/v1/tasks

{
  "title": "Разовая задача",
  "scheduled_for": "2026-04-10"
}

## Принятые решения

### 1. Генерация задач вперёд

Выбран горизонт 30 дней:
Упрощает реализацию;
Не перегружает БД.

### 2. Хранение recurrence отдельно

Позволяет:
Гибко расширять логику;
Не усложнять таблицу tasks.

### 3. JSONB для specific_dates

Позволяет хранить массив дат без отдельной таблицы.

### 4. template_id nullable

Позволяет:
Отличать обычные задачи от периодических;
Не ломать существующий функционал.

## Ограничения

Нет фонового обновления задач (cron);
Генерация только при создании шаблона;
Нет обновления уже сгенерированных задач при изменении шаблона.
